## Purpose

Ensure that analytics and session history features correctly display category names (rather than raw IDs or null values) by populating `Question.categoryId` at question creation time, AND compute accurate score aggregates that reflect only the latest evaluation per answer across all question types.

## Requirements

### Requirement: Question entities SHALL have categoryId populated at creation time
When questions are created via `POST /interview/start`, each persisted `Question` entity MUST have its `categoryId` field set to the resolved comma-separated category ID string derived from topic names returned by the AI. If the AI response contains no topics, categoryId MAY remain null.

#### Scenario: Question created with topic names
- **WHEN** a question is generated by the AI and the response includes non-empty `topics` array (e.g., `["Collections", "Streams"]`)
- **THEN** `QuestionGenerationService` resolves each topic name to its category ID via `InterviewSetupService.resolveTopicNamesToIds()`, and sets `question.setCategoryId(resolvedIdString)` before persisting

#### Scenario: Question created without topics in AI response
- **WHEN** the AI response contains an empty or null `topics` array
- **THEN** `categoryId` remains null; no exception is thrown, and the question is persisted normally

### Requirement: Analytics page SHALL display category names in session history table
The analytics page's session history table MUST render actual category name strings (e.g., "Collections", "Streams") in the Category column by binding to the correct DTO field (`categoryName`, not `categoryId`).

#### Scenario: Session row renders category name from backend
- **WHEN** a completed session has questions with non-null categoryId values
- **THEN** the SQL query uses `string_to_array + unnest + EXISTS` to join multi-value `questions.category_id` against `categories.id` and returns the category names, which is bound as `session.categoryName` in the HTML template

#### Scenario: Session row without valid category falls back gracefully
- **WHEN** a session has no questions with populated categoryId (e.g., pre-existing rows or AI omitting topics)
- **THEN** the Category column displays "N/A" instead of crashing or showing undefined

### Requirement: Analytics page SHALL display accurate per-category average scores
The Category Breakdown bar chart MUST reflect only the latest evaluation per answer and must correctly include questions tagged with multiple categories. Each question's score contributes to every category it is tagged under via `string_to_array + unnest + EXISTS` join.

#### Scenario: Single-category question contributes to its category
- **WHEN** a completed session contains a theory or code question with `categoryId = "11"` (e.g., Core Java) and the answer has been evaluated once with score 8
- **THEN** the Category Breakdown chart shows Core Java with an average score of 8

#### Scenario: Multi-category question contributes to all tagged categories
- **WHEN** a completed session contains a question with `categoryId = "11,13"` (Core Java + Streams) and the answer has been evaluated once with score 7
- **THEN** both Core Java and Streams show score 7 in their respective bars — the question is counted under each category it belongs to

#### Scenario: Retried answers use only latest evaluation score
- **WHEN** a user retries an answer, generating a new Evaluation record (the old FAILED record remains in DB)
- **THEN** the Category Breakdown averages only the latest evaluation per answer — stale retry scores do not pollute the average

### Requirement: Analytics page SHALL display correct overall and session-level average scores
The global "Average Score" metric card AND each row's score in the session history table MUST compute `AVG(e.score)` over all evaluations for completed sessions, considering only the latest evaluation per answer. The session history score column MUST show total score (`avg × questionCount`) divided by possible points (`questionCount × 10`).

#### Scenario: Global average reflects all question types
- **WHEN** a user has completed sessions containing both THEORY and CODE questions with evaluated answers
- **THEN** the global "Average Score" metric averages across ALL evaluated answers regardless of type — no exclusion of THEORY questions

#### Scenario: Session-level score includes THEORY and CODE questions
- **WHEN** a session contains 3 code questions (avg score 7) and 2 theory questions (avg score 9)
- **THEN** the session row displays an average reflecting all five question types (approximately 8), not just the code questions

#### Scenario: Session history numerator shows total, not per-question average
- **WHEN** a session has `score = 7` (per-question average) and `questionCount = 3`
- **THEN** the score column displays "21/30" — i.e., `(score × questionCount)` over `(questionCount × 10)`

#### Scenario: Retried answers use only latest evaluation for global and session averages
- **WHEN** a user has completed sessions where some answers were retried (multiple Evaluation records per Answer)
- **THEN** both the global Average Score metric and each session's score in the history table count only the latest evaluation per answer — stale retry scores do not pollute either average
